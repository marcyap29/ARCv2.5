# Bug Tracker Template

## Bug Report Format

### Bug ID: BUG-YYYY-MM-DD-###
**Title**: Brief descriptive title

**Severity**: Critical | High | Medium | Low  
**Priority**: P1 (Blocker) | P2 (High) | P3 (Medium) | P4 (Low)  
**Status**: Open | In Progress | Fixed | Verified | Closed  
**Reporter**: Name/Role  
**Assignee**: Developer  
**Found Date**: YYYY-MM-DD  
**Fixed Date**: YYYY-MM-DD (if applicable)  

#### Description
Clear description of the bug behavior

#### Steps to Reproduce
1. Step one
2. Step two  
3. Step three

#### Expected Behavior
What should happen

#### Actual Behavior
What actually happens

#### Environment
- Device: iPhone 16 Pro Simulator / Physical device
- OS: iOS 18.0
- Flutter Version: x.x.x
- App Version: x.x.x

#### Root Cause
Technical explanation of why the bug occurred

#### Solution
Technical description of the fix implemented

#### Files Modified
- `path/to/file1.dart`
- `path/to/file2.dart`

#### Testing Notes
How the fix was validated

#### Related Issues
Links to related bugs or tickets

---

## Severity Definitions

- **Critical**: App crashes, data loss, security issues, core functionality broken
- **High**: Major feature not working, significant UX problems, performance issues
- **Medium**: Minor feature issues, UI glitches, edge cases
- **Low**: Cosmetic issues, nice-to-have improvements

## Priority Definitions  

- **P1 (Blocker)**: Prevents release, must fix immediately
- **P2 (High)**: Important for user experience, fix in current sprint
- **P3 (Medium)**: Should fix when time permits
- **P4 (Low)**: Enhancement or minor issue

## Status Workflow

Open → In Progress → Fixed → Verified → Closed
